# Проект «Telegram-бот для управления ту-ду-листом»

## Описание
Бот для Telegram, позволяющий пользователям управлять списком задач: добавлять, просматривать, 
отмечать как выполненные, архивировать и удалять. Данные хранятся в базе данных PostgreSQL.

## Установка
### 1. Установите Python (рекомендуется версия 3.6+).

### 2. Установите зависимости:
```
pip install telebot psycopg2
```

### 3. Настройте конфигурацию в файле config.py:
```
BOT_TOKEN = 'YOUR_BOT_TOKEN'
HOST = 'localhost'
DB = 'task_db'
USER = 'postgres'
PAS = 'your_password'
```

### 4. Запустите бота:
python main.py

## Использование
### Доступные команды
```
/start — запустить бота и получить приветственное сообщение.
/help — показать список доступных команд.
/add <название> — добавить новую задачу (пример: /add Купить молоко).
/list — вывести список активных (неархивированных) задач.
/archive — показать архивные задачи.
/complete <номер> — отметить задачу как выполненную (пример: /complete 68).
/delete <номер> — «удалить» задачу (на самом деле помечает её как архивированную).
```

## Архитектура
Бот построен на следующих компонентах:

- Телебот: используется библиотека telebot для обработки сообщений Telegram.
- База данных: PostgreSQL с таблицей tasks, содержащей:
  - id — уникальный идентификатор задачи;
  - name — название задачи;
  - is_completed — флаг выполнения (по умолчанию FALSE);
  - is_archived — флаг архивации (по умолчанию FALSE);
  - user_id — идентификатор пользователя Telegram.

## Обработка ошибок
При ошибках добавления/обновления задачи бот отправляет сообщение об ошибке и откатывает транзакцию.
Некорректные команды обрабатываются стандартным сообщением с предложением использовать /help.

## Что можно изменить
### 1. Улучшение обработки удаления
Сейчас команда ```/delete``` лишь помечает задачу как ```is_archived=TRUE```. Можно реализовать полное удаление из БД с проверкой прав пользователя.

### 2. Оптимизация работы с БД:
- Использовать контекстные менеджеры (with) для автоматического закрытия соединений и курсоров.
- Внедрить пул соединений для масштабируемости.
- Заменить прямые SQL-запросы на ORM (например, SQLAlchemy) для упрощения работы с данными.

### 3. Безопасность и конфигурация:
- Перенести конфиденциальные данные (токен бота, параметры БД) в переменные окружения вместо файла config.py.
- Добавить проверку прав доступа (например, запретить изменять чужие задачи).

### 4. Обработка исключений:
- Заменить общий ```except Exception as e``` на конкретные исключения (например, ```psycopg2.DatabaseError```).
- Реализовать механизм восстановления соединения с БД при временных сбоях.

## Что добавить в будущем
### 1. Функциональность редактирования
- Команда ```/edit <id> <новое_название>``` для изменения названия задачи.
- Возможность обновлять сроки, теги или описание задачи.

### 2. Управление сроками
- Добавить поле ```due_date``` (дата выполнения) в таблицу ```tasks```.
- Реализовать команду ```/set_due <id> <дата>```.
- Настроить напоминания о приближающихся дедлайнах.

### 3. Расширенные фильтры и сортировка
- Фильтрация задач по статусам (активные, выполненные, архивированные).
- Сортировка по дате создания, сроку выполнения, алфавиту.
- Команда ```/search <ключевое_слово>``` для поиска задач по названию.

### 4. Улучшение интерфейса
- Использовать inline-клавиатуры для выбора задач вместо ввода ID вручную.
- Форматировать вывод списка задач с эмодзи, цветами или Markdown.
- Добавить возможность прикреплять файлы (например, чек-листы к задачам).

### 5. Веб-интерфейс
- Создать веб-приложение, синхронизированное с БД, для управления задачами вне Telegram.

### 6. Локализация
- Реализовать поддержку нескольких языков (например, русский, английский) через словари переводов.

### 7. Тестирование
- Написать юнит-тесты для:
  - добавления/обновления/удаления задач;
  - обработки некорректных команд;
  - взаимодействия с БД.

### 8. Логирование
Внедрить систему логирования (например, через ```logging``` модуль) для:
- отслеживания запросов пользователей;
- записи ошибок и предупреждений;
- мониторинга производительности.

### 9. Масштабирование
Рассмотреть переход на асинхронный фреймворк (например, ```aiogram```), чтобы обрабатывать тысячи запросов без блокировки.

### 10. Дополнительные возможности
- Добавление тегов к задачам (например, ```/add Купить молоко #продукты```).
- Совместный доступ (добавление задач для других пользователей).
- Экспорт/импорт списка задач в CSV/JSON.
